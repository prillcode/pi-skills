# Phase 03: Session Management - Plan

## Objective

Create a Session Management panel for viewing, switching between, and bookmarking pi sessions.

## Context

**Reference:**
- @file /home/prill/.nvm/versions/node/v22.20.0/lib/node_modules/@mariozechner/pi-coding-agent/examples/extensions/preset.ts - Shows SelectList pattern
- @file /home/prill/dev/pi-skills/.planning/pcw-01-custom-pi-tui/BRIEF.md - Architecture decisions
- Pi API: `SessionManager.list()`, `SessionManager.listAll()`, `ctx.switchSession()`

**Key Insight:**
Unlike embedding a full editor (which requires PTY/terminal emulation), session management uses pi's built-in session APIs. This is fully supported and stable.

## Session Data Available

- Session file path
- Creation time
- Message count
- Token usage
- Custom labels/bookmarks (via `pi.setLabel()`)

## Tasks

### Task 1: Create sessions utility module

**Files:** `~/dev/pi-skills/extensions/dashboard/utils/sessions.ts`

**Interface:**
```typescript
import { SessionManager } from "@mariozechner/pi-coding-agent";

export interface SessionInfo {
  file: string;
  name: string;
  created: Date;
  messageCount: number;
  tokenCount: number;
  cost: number;
  isBookmarked: boolean;
}

export async function listSessions(cwd: string): Promise<SessionInfo[]>;
export async function getSessionStats(file: string): Promise<{
  messageCount: number;
  inputTokens: number;
  outputTokens: number;
  cost: number;
}>;
export function formatSessionName(file: string): string;
```

**Verify:**
- Module exports all functions
- Properly uses SessionManager API
- Handles missing/invalid sessions gracefully
