# Phase 01: Foundation - Plan

## Objective

Move the existing dashboard extension from `~/.pi/agent/extensions/dashboard.ts` to the `pi-skills` repo with proper structure, and set up symlinks for pi to load it.

## Context

**Existing Code:**
- Current dashboard at: `~/.pi/agent/extensions/dashboard.ts`
- Target location: `~/dev/pi-skills/extensions/dashboard/index.ts`

**Desired Structure:**
```
extensions/
└── dashboard/
    ├── index.ts          # Main entry point
    ├── components/       # UI components
    │   └── Dashboard.ts  # Extract dashboard component
    ├── utils/            # Utility functions
    │   └── format.ts     # Formatting helpers
    └── README.md
```

## Tasks

### Task 1: Create directory structure

**Files:** Create directories

```bash
mkdir -p ~/dev/pi-skills/extensions/dashboard/components
mkdir -p ~/dev/pi-skills/extensions/dashboard/utils
```

**Verify:** `ls -la ~/dev/pi-skills/extensions/dashboard/`

---

### Task 2: Move and refactor main extension

**Files:**
- Read: `~/.pi/agent/extensions/dashboard.ts`
- Write: `~/dev/pi-skills/extensions/dashboard/index.ts`

**Actions:**
1. Move main extension code to `index.ts`
2. Extract `DashboardComponent` class to separate file
3. Extract utility functions to `utils/format.ts`
4. Update imports

**Verify:**
- `cat ~/dev/pi-skills/extensions/dashboard/index.ts` shows clean entry point
- No TypeScript errors in structure

---

### Task 3: Create symlink

**Files:** Symlink setup

**Actions:**
```bash
# Remove existing file if present
rm -f ~/.pi/agent/extensions/dashboard.ts

# Create symlink to directory
ln -s ~/dev/pi-skills/extensions/dashboard ~/.pi/agent/extensions/dashboard
```

**Verify:** `ls -la ~/.pi/agent/extensions/ | grep dashboard`

---

### Task 4: Test the extension

**Actions:**
1. Run `pi` in test mode with extension
2. Test `/dashboard` command
3. Test `/todo` command
4. Test `/footer` command

**Verify:**
- Dashboard opens without errors
- All tabs visible
- Todos add/complete work
- Footer toggles

---
